Privacy-aware LLM masking for internal-data readiness

사내/민감 데이터를 LLM 분석에 활용하기 전에, 원문 값을 외부 모델에 직접 보내지 않는 경계를 합성 예시로 시연한 미니 프로젝트입니다.

문제

Raw prompt risk

업무 텍스트에는 이름, 계좌형 문자열, 이메일, 주소, 금액, 일정이 섞입니다.

Naive masking

무의미한 토큰은 안전하지만 문맥을 잃어 번역/요약 품질이 흔들립니다.

Restore fragility

모델이 토큰이나 surrogate를 변형하면 원복이 깨집니다.

권장 구조

{AMOUNT_0}
{BANK_ACCOUNT_0}
{ACCOUNT_HOLDER_0}
{DEADLINE_0}
{EMAIL_0}
{ADDRESS_0}

외부 모델에 보낼 텍스트는 placeholder와 역할 설명만 남기고, 원문 값은 로컬 매핑으로만 복원합니다.

흐름

1
Detect spans
2
Replace with semantic placeholders
3
Prepare metadata hints only
4
Validate placeholders
5
Restore locally

경계

이 데모는 합성 예시만 사용하며, 실제 사내 데이터나 고객/프로젝트 식별자는 포함하지 않습니다. 외부 API를 호출하지 않고 모델 단계는 시뮬레이션합니다. 목적은 규정 준수 제품을 주장하는 것이 아니라, LLM 활용 전 마스킹 사고방식을 보여주는 것입니다.
